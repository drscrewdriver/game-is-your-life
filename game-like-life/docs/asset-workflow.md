# GameLife 资产生成工作流

> **状态**: v0.2 草案（加入 Wan 视频生成能力）
> **生成工具**: Wan 2.1（万相）图+视频双引擎 → 不及预期则切换 Google / OpenAI
> **技术约束**: 独立 HTML 文件，无构建工具，CDN 加载 Tailwind + Chart.js
> **资产形态**: 静态图（PNG）+ 短视频（WebM/MP4）双轨并行

## 一、工具链与优先级

### 1.1 图像生成

| 优先级 | 工具 | 适用场景 | 备注 |
|--------|------|---------|------|
| P0 | **Wan 2.1 t2i** | Q版形象静态图、概念图 | 阿里云百炼平台，1024×1024 |
| P1 | **Google Imagen 3** | 精细风格一致性要求 | Vertex AI API，风格锁定强 |
| P2 | **OpenAI gpt-image-1** | 备选 | 语义理解好，Q版风格需反复调 |

### 1.2 视频生成

| 优先级 | 工具 | 适用场景 | 备注 |
|--------|------|---------|------|
| P0 | **Wan 2.1 t2v** | 角色动态头像、待机动画 | 图生视频(i2v) 或 文生视频(t2v) |
| P1 | **Google Veo 2** | 高质量短视频备选 | 风格可控性好，API 较贵 |
| P2 | **OpenAI Sora** | 备选 | 语义理解强，风格一致性需调 |

### 1.3 后处理

| 工具 | 用途 | 备注 |
|------|------|------|
| **remove.bg / rembg** | 静态图抠图去背景 | 必须透明底 |
| **FFmpeg** | 视频裁剪/压缩/转码/加透明通道 | 视频后处理核心工具 |
| **TinyPNG / squoosh** | 静态图压缩 | 控制单页资产总量 |
| **HandBrake / FFmpeg** | 视频压缩（CRF 调参） | 控制单视频 < 200KB |
| **Sprite Sheet 合成** | 从视频抽帧 → 拼精灵表 | FFmpeg 抽帧 + ImageMagick 拼图 |

### 1.4 静态图 vs 视频 — 使用决策矩阵

| 使用场景 | 推荐形态 | 理由 |
|----------|---------|------|
| **角色头像（导航栏/卡片）** | 🎬 视频 | 微动表情/呼吸，提升「活着感」 |
| **角色立绘（详情页）** | 🖼️ 静态 | 大图不需要动，加载快 |
| **行走/移动动画** | 🖼️ Sprite Sheet | 精确帧控、性能好、方向切换灵活 |
| **技能释放/升级特效** | 🎬 视频 | 粒子/光效视频表现力远超精灵表 |
| **场景背景** | 🖼️ 静态 | 背景不需要动 |
| **UI 元素（徽章/按钮）** | 🖼️ 静态 | 小元素不需要视频 |
| **成就/徽章动效** | 🎬 视频 | 获得成就时的庆祝动画 |

## 三、Q版形象 — 技术规格

### 3.1 生成参数

| 参数 | 规格 | 说明 |
|------|------|------|
| **生成分辨率** | 1024 × 1024 px | Wan 原生输出尺寸 |
| **最终使用尺寸** | 128×128 / 64×64 / 32×32 | 三档，覆盖头像/图标/迷你 |
| **格式** | PNG-24（透明底） | 必须去背景 |
| **色彩模式** | sRGB | Web 标准 |
| **文件大小** | 单张 < 50KB（128px档） | squoosh 压缩后 |

### 3.2 人体比例体系

Q版的核心是「夸张头身比 + 简化躯干」，必须全角色统一：

```
┌─────────────────────────────────────────────────┐
│  标准 2 头身比例（以 128×128 输出为基准）         │
│                                                   │
│  ┌──────────┐  ← 头部区域：64×64px（占总面积50%） │
│  │  头发区   │     发顶到下巴 = 头高               │
│  │ ┌──────┐ │                                     │
│  │ │ 脸部  │ │     眼睛位于头高 50% 线（非写实40%） │
│  │ │ 五官  │ │     眼距 = 1只眼宽                   │
│  │ └──────┘ │     嘴 = 1条弧线，无需细节            │
│  └──────────┘                                     │
│    ┌──────┐  ← 身体区域：64×64px                   │
│    │ 躯干  │     肩宽 = 头宽 × 0.7                 │
│    │ 四肢  │     手臂 = 头高 × 0.5（可省略关节）    │
│    │ 脚部  │     腿 = 头高 × 0.4（短腿，脚占1/3）   │
│    └──────┘                                       │
│                                                   │
│  总高 = 2 头身 ± 0.2 头（头发装饰可超出）          │
└─────────────────────────────────────────────────┘
```

| 部位 | 比例规则 | 简化要点 |
|------|---------|----------|
| **头部** | 占全身 45-50%，圆形/椭圆 | 最大信息密度区域，表情/职业辨识核心 |
| **眼睛** | 占头高 25-30%，大而有神 | 高光点 1-2 个，瞳孔含元素主色 |
| **嘴巴** | 极简 1 线条 | 微笑/严肃/惊讶，3 种基础形态即可 |
| **鼻子** | 可省略或仅 1 小点 | Q版不画完整鼻子 |
| **躯干** | 梯形/矩形，无腰线 | 用颜色/纹章区分职业，不画肌肉 |
| **手臂** | 圆柱体，1-2 节 | 手部可简化为圆形手套/拳头 |
| **腿部** | 短粗圆柱 | 鞋子是职业辨识辅助元素 |

### 3.3 五官与表情系统

每个角色需要 **3 种基础表情**，用于不同状态：

| 表情 | 使用场景 | 五官变化 |
|------|---------|----------|
| **默认（自信微笑）** | 头像、idle 状态 | 嘴角上扬弧线，眼睛正常 |
| **专注（认真）** | 任务执行中、walk 状态 | 嘴角平直，眉毛微压 |
| **开心（庆祝）** | 升级、任务完成 | 嘴巴张开/大笑弧线，眼睛弯月形 |

**五官绘制规则**：
- 眼睛形状统一为「大圆 + 下方高光」，不同职业仅改瞳孔颜色
- 眉毛 = 2px 粗弧线，角度传达情绪（上扬=开心，下压=专注）
- 所有表情在 64×64 缩小后仍可区分

### 3.4 元素色系规范

与项目「IT职业魔法化映射」对齐，每个元素有严格色板：

| 元素 | 主色 | 辅色 | 瞳孔色 | 代表职业 | Hex 色板 |
|------|------|------|--------|---------|----------|
| 🔥 火系 | 红橙 | 金黄 | #E25600 | 后端 | `#EF4444` `#F97316` `#FBBF24` |
| 🌬️ 风系 | 青蓝 | 白 | #06B6D4 | 前端 | `#06B6D4` `#67E8F9` `#E0F2FE` |
| ⚡ 雷系 | 紫 | 银白 | #7C3AED | Go 语言 | `#7C3AED` `#A78BFA` `#EDE9FE` |
| 🌍 土系 | 棕绿 | 米黄 | #84CC16 | Python | `#84CC16` `#A3E635` `#ECFCCB` |
| 💧 水系 | 深蓝 | 冰蓝 | #3B82F6 | SQL/数据 | `#3B82F6` `#93C5FD` `#DBEAFE` |
| 🔮 奥术 | 深紫 | 星金 | #6366F1 | C/底层 | `#6366F1` `#818CF8` `#E0E7FF` |

**上色规则**：
- 赛璐珞 2 色法：主色 + 暗部（主色加深 20%），不加渐变
- 元素色仅用于：装备主色、瞳孔、技能特效光晕
- 服装底色统一为中性色（白/灰/深灰），避免与元素色冲突

### 3.6 多视图要求

每个角色最终需产出 **3 视图**（用于精灵表制作）：

| 视图 | 用途 | 生成方式 |
|------|------|----------|
| **正面** | 头像、idle/walk-down | 直接生成 |
| **左侧面** | walk-left | 生成后水平翻转得右侧 |
| **背面** | walk-up | 直接生成，注意头发/背部装备细节 |

> 💡 左右对称角色可只生成正面 + 背面，左侧 = 正面镜像。非对称角色（单肩甲、斜挎包等）必须单独生成左侧。

### 3.7 等级视觉进化规则

角色升级时形象不需要重绘，但通过「叠加装饰」体现成长：

| 等级段 | 视觉变化 | 实现方式 |
|--------|---------|----------|
| Lv.1-9 | 基础形象 | 原始立绘 |
| Lv.10-19 | + 肩甲/披风 | 叠加图层（overlay PNG） |
| Lv.20-29 | + 武器升级 + 光环 | 更换武器图层 + 背后光效 |
| Lv.30+ | + 翅膀/冠冕 + 全身光效 | 全套叠加图层 |

> 这样避免每级重绘，只需准备 **基础体 + 装饰图层**，组合即可。

### 3.8 风格一致性控制

| 手段 | 方法 |
|------|------|
| **种子锁定** | 先用 Wan 生成一个基准角色，记录 seed 值，后续角色用同 seed 微调 |
| **参考图引导** | 若 Wan 支持 img2img，将基准角色作为风格参考图输入（weight 0.6-0.8） |
| **负面提示词** | 统一排除（见 3.5），确保不会出现写实/3D 偏移 |
| **人工校验** | 每个角色生成 4-8 张候选，人工挑选最符合基准风格的 |
| **色板校验** | 用取色器检查元素色是否在规范 Hex 范围内，偏差 > 15% 需重新生成 |

### 3.9 质量检查清单

- [ ] 头身比一致（全部角色 2~2.5 头身，误差 < 5%）
- [ ] 线稿粗细统一（2px @1024，缩放后等比）
- [ ] 赛璐珞 2 色法上色，无渐变、无厚涂
- [ ] 透明背景干净，无残边、无白边
- [ ] 缩小到 64×64 后五官仍可辨识
- [ ] 缩小到 32×32 后职业仍可区分（靠剪影 + 主色）
- [ ] 同色系角色之间有色差区分度（主色 Hex 差 > 30）
- [ ] 3 种表情在 64px 下可区分
- [ ] 正面/侧面/背面三视图风格一致
- [ ] 元素色在规范色板范围内
- [ ] 文件体积达标（< 50KB @128px，< 15KB @64px）
- [ ] 装饰叠加图层对齐基准体（像素级对齐检查）

## 四、位移动图 — 技术规格

### 4.1 技术方案选型

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| **CSS Sprite Sheet** ✅ 推荐 | 无依赖、性能好、控制精确 | 需手工拼精灵表 | 角色行走、动作动画 |
| **Animated GIF** | 简单直接 | 256色限制、体积大、无法精确控制帧 | 不推荐用于UI |
| **APNG** | 全彩、透明支持 | 文件偏大、部分旧浏览器不支持 | 特效动画（可选） |
| **Lottie JSON** | 矢量缩放、体积极小 | 需引入 lottie-web (~70KB) | 复杂UI动效（后期考虑） |

**结论**: 位移动图采用 **CSS Sprite Sheet + keyframes** 方案，零依赖，与独立 HTML 约束完全兼容。

### 4.2 运动物理学 — Q版角色行走规律

Q版角色的运动不是「缩小版真人走路」，需要夸张化处理：

```
行走周期关键帧分解（4帧 = 1个完整步态周期）：

  帧1: 接触帧        帧2: 下沉帧        帧3: 经过帧        帧4: 高位帧
  ┌──────┐          ┌──────┐          ┌──────┐          ┌──────┐
  │  ◯   │          │  ◯   │          │  ◯   │          │  ◯   │
  │ /│\  │          │ /│\  │          │  │   │          │ /│\  │
  │/ │   │          │  │   │          │ /│\  │          │/ │   │
  │  │   │          │ /│\  │          │/ │   │          │  │   │
  └──────┘          └──────┘          └──────┘          └──────┘
  前脚着地           身体最低           单脚站立           身体最高
  重心偏前           膝盖弯曲最大       重心居中           后腿蹬起
```

| 关键帧 | 身体高度 | 重心位置 | 四肢状态 | 头部晃动 |
|--------|---------|---------|---------|----------|
| **帧1 接触** | 中 | 偏前 60% | 前脚伸直，后脚翘起 | 微前倾 |
| **帧2 下沉** | 最低（-2px） | 居中 | 双膝弯曲，手臂下摆 | 回正 |
| **帧3 经过** | 中 | 居中偏后 | 单脚支撑，另一腿经过 | 微后仰 |
| **帧4 高位** | 最高（+2px） | 偏前 | 后腿蹬直，前腿抬起 | 微前倾 |

**Q版夸张化要点**：
- 上下弹跳幅度 = 帧高的 3-5%（64px帧 ≈ 2-3px），比写实风更大
- 手臂摆动简化为「前后一根棍」，无需画出手肘关节
- 头部运动滞后身体 0.5 帧（产生可爱的「摇头」感）
- 整体节奏感 > 物理准确性，追求「弹弹的」感觉

### 4.3 Sprite Sheet 规格

| 参数 | 规格 | 说明 |
|------|------|------|
| **单帧尺寸** | 64×64 px 或 128×128 px | 与静态头像档位对齐 |
| **方向数** | 4方向（下/左/右/上） | 标准 RPG 四方向 |
| **每方向帧数** | 4 帧（完整步态周期） | 接触→下沉→经过→高位 |
| **精灵表布局** | 横排：帧序号；纵排：方向 | 4列 × 4行 = 16帧/表 |
| **总表尺寸** | 256×256（64px帧）或 512×512（128px帧） | 单表 < 100KB |
| **格式** | PNG-24 透明底 | 同静态图 |

**精灵表布局示意（64px 帧）**：
```
         帧1(接触)  帧2(下沉)  帧3(经过)  帧4(高位)
         ┌────────┬────────┬────────┬────────┐
下行(0)  │  0,0   │  1,0   │  2,0   │  3,0   │  ← 面朝玩家
         ├────────┼────────┼────────┼────────┤
左行(1)  │  0,1   │  1,1   │  2,1   │  3,1   │  ← 面朝左
         ├────────┼────────┼────────┼────────┤
右行(2)  │  0,2   │  1,2   │  2,2   │  3,2   │  ← 面朝右
         ├────────┼────────┼────────┼────────┤
上行(3)  │  0,3   │  1,3   │  2,3   │  3,3   │  ← 背朝玩家
         └────────┴────────┴────────┴────────┘

坐标计算: x = frame × 64,  y = direction × 64
```

### 4.4 帧率与动画参数

| 动画类型 | FPS | 帧间隔 | 循环方式 | 弹跳幅度 |
|----------|-----|--------|---------|----------|
| **idle（待机）** | 2 FPS | 500ms/帧 | 无限循环 | 0-1px 微呼吸 |
| **walk（行走）** | 4 FPS | 250ms/帧 | 无限循环（移动期间） | 2-3px 弹跳 |
| **run（奔跑）** | 6 FPS | 167ms/帧 | 无限循环 | 3-4px 大弹跳 |
| **action（技能释放）** | 6 FPS | 167ms/帧 | 播放一次 | 无（原地） |
| **levelup（升级特效）** | 8 FPS | 125ms/帧 | 播放一次 | N/A（特效层） |

**idle 呼吸动画详解（2帧）**：
```
帧A: 正常站立，身体 y=0
帧B: 微吸气，身体 y=-1px，手臂微张
→ 效果：轻柔的「呼吸起伏」，不是跳跃
```

**walk → idle 过渡**：
- 不能直接跳切，需要 1 帧「刹车帧」（双脚并拢，身体回正）
- 过渡时间 = 250ms（1帧），视觉上自然减速

### 4.5 装备与附属物动态规则

角色身上的非刚性部件需要「跟随运动」，增加生动感：

| 附属物 | 运动规则 | 延迟帧 | 幅度 |
|--------|---------|--------|------|
| **披风/斗篷** | 滞后身体运动，行走时向后飘 | +1 帧 | 摆动 5-8° |
| **长发** | 滞后身体，幅度小于披风 | +0.5 帧 | 摆动 3-5° |
| **武器（手持）** | 随手臂摆动，但幅度更小 | 同步手臂 | 摆动 2-3° |
| **帽子/头饰** | 弹跳时微翘 | 同步身体 | 上下 1px |
| **翅膀/尾翼** | 独立扇动，频率高于步频 | 独立节奏 | 扇动 10-15° |
| **光效/粒子** | 跟随角色位置，独立粒子运动 | N/A | CSS 独立动画 |

**实现方式**：
- 披风/头发/武器 → 画入精灵表帧内（手绘时直接画在不同帧的位置）
- 光效/粒子 → CSS 独立图层叠加，`position: absolute` 对齐角色

### 4.6 方向绘制细节

| 方向 | 可见内容 | 关键辨识点 | 绘制注意 |
|------|---------|-----------|----------|
| **下行（面朝玩家）** | 正脸、全身正面 | 表情、正面装备 | 最重要的方向，细节最多 |
| **左行** | 左侧面、左耳/左眼 | 侧脸轮廓、武器手持侧 | 非对称角色需单独绘制 |
| **右行** | 右侧面 | 与左行镜像 | 对称角色可直接水平翻转左行 |
| **上行（背朝玩家）** | 背影、背部装备 | 发型背面、披风/翅膀 | 需体现背部装备细节 |

**左右镜像判断规则**：
- 角色装备完全对称（双肩甲、双手武器）→ 可镜像
- 角色装备非对称（单肩甲、斜挎包、左手盾牌）→ 必须单独绘制右行

### 4.7 CSS 实现模板

```
/* ===== 精灵表动画引擎 ===== */
.sprite {
  width: 64px;
  height: 64px;
  background-image: url('assets/characters/sprites/warrior-sprite.png');
  background-repeat: no-repeat;
  image-rendering: pixelated;       /* 像素风锐利缩放 */
  image-rendering: -moz-crisp-edges; /* Firefox 兼容 */
  will-change: background-position;  /* GPU 加速提示 */
}

/* --- idle 呼吸动画（2帧循环，下行方向） --- */
.sprite.idle-down {
  background-position-y: 0;         /* 第0行 = 下行 */
  animation: idle-breathe 1s steps(2) infinite;
}
@keyframes idle-breathe {
  from { background-position-x: 0; }       /* 帧1: 接触 */
  to   { background-position-x: -64px; }   /* 帧2: 下沉 */
}

/* --- walk 行走动画（4帧循环） --- */
.sprite.walk-down {
  background-position-y: 0;
  animation: walk-cycle 1s steps(4) infinite;
}
.sprite.walk-left {
  background-position-y: -64px;
  animation: walk-cycle 1s steps(4) infinite;
}
.sprite.walk-right {
  background-position-y: -128px;
  animation: walk-cycle 1s steps(4) infinite;
}
.sprite.walk-up {
  background-position-y: -192px;
  animation: walk-cycle 1s steps(4) infinite;
}
@keyframes walk-cycle {
  0%   { background-position-x: 0; }        /* 帧1: 接触 */
  25%  { background-position-x: -64px; }    /* 帧2: 下沉 */
  50%  { background-position-x: -128px; }   /* 帧3: 经过 */
  75%  { background-position-x: -192px; }   /* 帧4: 高位 */
  100% { background-position-x: 0; }        /* 循环回帧1 */
}

/* --- run 奔跑（同 walk 但更快） --- */
.sprite.run-down { animation-duration: 0.67s; }  /* 6FPS 节奏 */

/* --- 过渡：walk → idle 刹车帧 --- */
.sprite.brake {
  animation: brake-stop 0.25s steps(1) forwards;
  /* 停在帧1（接触帧）作为刹车 */
}
@keyframes brake-stop {
  to { background-position-x: 0; }
}
```

```javascript
// JS 方向切换辅助函数
function setDirection(spriteEl, direction, action) {
  // 清除旧状态
  spriteEl.className = 'sprite';
  // 设置新方向 + 动作
  spriteEl.classList.add(`${action}-${direction}`);
  // direction: 'down' | 'left' | 'right' | 'up'
  // action: 'idle' | 'walk' | 'run'
}

// 示例：角色开始向左走
setDirection(character, 'left', 'walk');
// 示例：角色停下待机
setDirection(character, 'left', 'idle');
```

### 4.8 从静态图生成精灵表的工作流

```
┌──────────────────────────────────────────────────────────────┐
│  输入: 1张已选定的角色正面静态图（1024×1024 去背景）           │
├──────────────────────────────────────────────────────────────┤
│                                                               │
│  Step A: 生成 4 方向静态图                                    │
│  ├─ 正面: 已有（输入图）                                      │
│  ├─ 背面: Wan 生成（prompt 加 "back view, same character"）   │
│  ├─ 左面: Wan 生成（prompt 加 "left side view"）              │
│  └─ 右面: 对称角色=左面镜像 / 非对称=Wan 生成                  │
│                                                               │
│  Step B: 每个方向生成 4 帧步态变体                             │
│  ├─ 用 Wan img2img + 姿态描述 prompt                          │
│  ├─ 帧1: "standing, one foot forward" (接触)                  │
│  ├─ 帧2: "crouching slightly, knees bent" (下沉)              │
│  ├─ 帧3: "mid-step, one leg raised" (经过)                    │
│  └─ 帧4: "standing tall, push off" (高位)                     │
│  × 4方向 = 共 16 帧（含正面已有 = 需新生成 15 帧）             │
│                                                               │
│  Step C: 统一后处理                                           │
│  ├─ 全部去背景                                                │
│  ├─ 统一缩放至单帧尺寸（64×64 或 128×128）                    │
│  ├─ 对齐角色重心点（所有帧中角色脚底位置一致）                  │
│  └─ 压缩                                                      │
│                                                               │
│  Step D: 拼合精灵表                                           │
│  ├─ 使用 ImageMagick / Canvas 脚本自动拼接                     │
│  ├─ 按布局规范排列（横=帧，纵=方向）                           │
│  └─ 输出最终 PNG                                               │
│                                                               │
│  Step E: 编写 CSS keyframes + 集成测试                         │
│  └─ 在原型 HTML 中验证动画效果                                 │
│                                                               │
└──────────────────────────────────────────────────────────────┘
```

### 4.9 质量检查清单

- [ ] 精灵表帧序正确，无错位（用 CSS 逐帧检查）
- [ ] 四方向动作对称性合理（左右镜像检查，非对称角色不镜像）
- [ ] 步态周期流畅：帧1→2→3→4→1 无跳帧、无抖动
- [ ] idle 呼吸自然，不像是跳跃
- [ ] walk → idle 过渡有刹车帧，不突然停止
- [ ] 重心对齐：所有帧中角色「脚底锚点」在同一像素位置
- [ ] 附属物（披风/头发/武器）跟随运动自然，延迟合理
- [ ] `image-rendering: pixelated` 下无模糊、无锯齿
- [ ] 缩小到 64px 后动画仍可感知（弹跳/摆动可见）
- [ ] 动画期间 CPU 占用合理（DevTools Performance < 5%）
- [ ] 精灵表总文件 < 100KB
- [ ] 所有方向 × 所有动作组合测试通过（4×3 = 12 种状态）

## 八、Wan 调用备忘

### 8.1 图像生成

| 项目 | 说明 |
|------|------|
| **平台** | 阿里云百炼（DashScope） |
| **模型** | wanx2.1-t2i |
| **API** | `POST /api/v1/services/aigc/text2image/image-synthesis` |
| **关键参数** | `size: 1024*1024`, `n: 4`, `seed: {固定值}` |
| **风格控制** | prompt + negative_prompt + seed 三件套 |

### 8.2 视频生成

| 项目 | 说明 |
|------|------|
| **模型** | wan2.1-i2v（图生视频）/ wan2.1-t2v（文生视频） |
| **API** | `POST /api/v1/services/aigc/image2video/video-synthesis` |
| **输入** | 静态角色图（i2v）或 文字描述（t2v） |
| **输出** | MP4 格式，时长 3-5s |
| **关键参数** | `resolution: 720*720`, `duration: 3-5`, `seed: {固定值}` |
| **角色动画 prompt** | "chibi character idle animation, subtle breathing, slight sway, loop-friendly, transparent background" |
| **行走动画 prompt** | "chibi character walking cycle, side view, 4 steps, consistent proportions, loop-friendly" |

### 8.3 后处理命令参考

```
# 视频抽帧（提取行走周期帧）
ffmpeg -i walk.mp4 -vf "fps=4" -frames:v 8 frame_%03d.png

# 视频转码为 WebM（透明通道支持）
ffmpeg -i input.mp4 -c:v libvpx-vp9 -b:v 200k -auto-alt-ref 0 output.webm

# 视频压缩（控制体积）
ffmpeg -i input.mp4 -c:v libx264 -crf 28 -preset slow -movflags +faststart output.mp4

# 视频裁剪（截取循环段）
ffmpeg -i input.mp4 -ss 0.5 -t 3.0 -c copy loop_segment.mp4

# 精灵表拼合
montage frame_*.png -tile 4x4 -geometry 64x64+0+0 -background none sprite-sheet.png

# 批量去背景（rembg CLI）
rembg p -m u2net input.png output.png
```
